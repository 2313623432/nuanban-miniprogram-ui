明白，你要的是这种逻辑：

**先有类目切换**，进入某个类目后，**下面是按时间倒序的记忆流**。
每条记忆里必须同时有：

1. **时间点**
2. **AI自然语言总结内容**

整体交互要像你给的 Replika memory：
**顶部切类目，下面上下滑动浏览该类目下不同时间点的记忆。**

下面我按这个思路，给你一版更准确的 **记忆模块 UI 设计稿说明**。

# 记忆模块 UI 设计稿

## 一、页面定位

这是一个 **用户记忆列表页**，用于查看 AI 在不同类目下记录的用户信息。

核心结构：

```text
顶部标题区
类目切换区
时间流式记忆列表
```

交互方式：

* 左右切换类目
* 上下滑动查看该类目下不同时间的记忆
* 每条记忆支持编辑、删除
* 支持新增记忆到当前类目

---

# 二、页面整体结构

## 1. 顶部导航区

顶部展示：

* 返回按钮
* 页面标题：`我的记忆`
* 右上角两个按钮：

  * `编辑`
  * `+`

结构示意：

```text
< 返回         我的记忆         编辑   ＋
```

说明：

* **编辑**：进入批量管理状态，可多选删除
* **+**：新增一条记忆，默认进入当前类目

---

## 2. 类目切换区

位于标题下方，横向滑动切换。

类目如下：

```text
身份信息   健康状况   生活习惯   养生目标
情绪状态   亲友情况   交流偏好   兴趣爱好
```

样式建议：

* 当前选中类目为高亮胶囊按钮
* 未选中类目为浅底按钮
* 支持左右滑动

例如当前选中：

```text
[健康状况]
 身份信息   生活习惯   养生目标 ...
```

---

# 三、记忆列表区

## 1. 展示逻辑

当前类目选中后，下方展示 **该类目下的所有记忆**。

排序方式：

```text
最新在上
最旧在下
```

即按 **时间倒序** 排列。

每条记忆都包含两个核心信息：

* **时间点**
* **内容**

---

## 2. 单条记忆卡片结构

每条记忆为一个圆角卡片，结构如下：

```text
时间
AI自然语言总结内容
```

示意：

```text
今天

你最近经常提到胃胀的问题，
尤其是在晚饭之后更明显。
```

再例如：

```text
3天前

你有轻度高血压，
平时会比较关注血压变化。
```

---

## 3. 卡片交互

点击单条记忆卡片：

进入该记忆的详情编辑页，支持：

* 编辑内容
* 删除记忆
* 修改所属类目

长按卡片（可选）：

* 弹出操作菜单

  * 编辑
  * 删除

---

# 四、时间显示规则

这个模块里的时间必须明确展示在每条记忆上方，不能只显示内容。

统一规则如下：

```text
刚刚
X分钟前
X小时前
昨天
前天
X天前
MM-DD
YYYY-MM-DD
```

例如：

```text
今天
昨天
前天
3天前
03-12
2024-11-05
```

为了更适合记忆模块，我建议这里比普通帖子更偏“日期感”，所以前端展示时可优先用：

* 今天
* 昨天
* 前天
* X天前
* MM-DD
* YYYY-MM-DD

这样用户会更容易理解“这条记忆是什么时候形成的”。

# 七、新增记忆交互

点击右上角 `+` 后：

进入新增记忆页。

页面结构：

```text
新增记忆

类目：健康状况
时间：默认今天
内容输入框

[保存]
```

说明：

* 默认带入当前类目
* 用户可改类目
* 保存后插入当前类目列表最上方

---

# 八、编辑记忆交互

点击某条记忆进入编辑页。

结构：

```text
编辑记忆

类目：健康状况
时间：昨天
内容：
你说这两天睡得不太安稳，
夜里容易醒。

[保存]   [删除]
```

说明：

* 时间默认不可编辑，表示记忆生成/更新时间
* 当用户修改内容后，该条记忆更新时间刷新到最新，排序置顶

# 九、UI视觉建议

这块一定要克制，不能做复杂配色。

## 1. 视觉层级

页面只保留三层信息：

* 标题/类目
* 时间
* 内容

## 2. 卡片风格

建议：

* 浅底色大圆角卡片
* 卡片之间留较大间距
* 时间小字放上方
* 内容为 2~4 行自然语言

## 3. 不建议做的

不要做：

* 每个类目不同颜色
* 过多图标
* 复杂标签堆叠
* 卡片里放太多按钮

这个模块的气质应该是：

> **安静、可信、像“AI替你整理出来的生活笔记”**

---

# 十、最终一句话定义

这个模块的正确形态应该是：

> **顶部按类目切换，底部按时间倒序浏览该类目下的自然语言记忆流。**

不是看板，不是表格，也不是多彩信息卡，而是 **“分类后的时间流记忆列表”**。
